# Security Policy

## Reporting a vulnerability

Please do not disclose suspected vulnerabilities in a public issue, discussion, pull request, commit, or social post.

Use GitHub's private vulnerability reporting flow instead:

1. Open this repository's **Security** tab.
2. Choose **Advisories** and then **Report a vulnerability**.
3. Include the affected area, reproduction steps, potential impact, and any suggested mitigation.

Please avoid including real credentials, personal data, or other sensitive production information in the report. Use redacted examples wherever possible.

If private vulnerability reporting is unavailable, open a public issue titled `Security contact request` without including vulnerability details. A maintainer can then arrange a private reporting channel.

We will acknowledge a private report when it is reviewed and coordinate disclosure after a fix or mitigation is available.
