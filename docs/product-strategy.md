# 盤勢：產品策略與使用者旅程

_2026-07-11 版_

## 1. 定位

> 看見一家公司的出生星圖，回看市場如何回應它的天象週期。

盤勢不是「星象叫多空」的交易訊號工具，也不是把一句每日運勢套到股票上。產品的三個支柱是：

1. **公司人格**：用成立、法人註冊或首次公開交易等生命事件建立命盤。
2. **市場天氣**：把行運窗口放到價格時間線，讓使用者親手探索。
3. **歷史驗證**：顯示全部樣本、失敗案例、報酬分布與資料限制，不只講「很準」的故事。

品牌比例是 **80% 清楚研究介面，20% 儀式感**。玄學負責讓人願意靠近，資料透明負責讓人願意留下。

### 最重要的市場缺口

直接競品不是過度像星座運勢，就是過度像 Bloomberg。尚缺少一個以「公司誕生事件」為中心，能在價格圖上探索、分享、質疑與實際回看的敘事型產品。繁體中文與臺股更幾乎沒有這個中間層。

## 3. 使用者與 JTBD

### 好奇型投資人

當一支股票暴漲、暴跌或長期盤整時，希望兩分鐘內看到一個不同於財報與技術面的時間敘事。成功不是得到買賣答案，而是找到值得繼續研究的日期與主題。

### 玄學熟悉、金融不熟者

懂水逆、行運與土星回歸，但看不懂密集金融圖表。希望用生活化語言理解公司命格與價格變化。

### 驗證型研究者

不接受只挑「很準」的例子。需要事件來源、算法、全部歷史樣本、樣本數、失敗案例與相對基準。

### 內容分享者

想分享一張有品味、有公司個性的圖，但不想分享看起來像推薦買股的海報。

### 反向人格

只想得到明確買／賣／持有答案的人不是盤勢要服務的主要使用者。

## 4. 首次使用旅程

```text
搜尋公司，不需登入
→ 確認交易所與時區
→ 選擇公司的「出生版本」
→ 0.8 秒起盤儀式
→ 公司命格印記
→ 價格 × 天象事件河流
→ 點一個事件看歷史樣本與反例
→ 存盤／分享／記住未來觀測窗口
```

### 關鍵節點

1. **首頁直接搜尋**：不用註冊，不詢問使用者的個人生辰。
2. **確認標的**：顯示公司、ticker、交易所與市場時區，避免同名或跨市場選錯。
3. **定生辰**：預設「首次公開交易」，旁邊立即顯示精度與來源。
4. **起盤揭幕**：600-900ms 星軌聚合成公司命格，支援 reduced motion，回訪時直接進結果。
5. **一句看懂**：先顯示核心質地與一句白話，再提供「為什麼這樣讀」。
6. **時間探索**：引導使用者點一個相位事件，價格、命盤與解讀同步定位。
7. **得到第一個 insight 後再邀請保存**：CTA 叫「加入觀測」或「記下這一刻」，不是「下單」。

## 5. 核心探索循環

搜尋公司 → 看公司命格 → 選日期／轉折 → 同步查看價格、天象與現實事件 → 展開解讀與依據 → 比較過往同類窗口 → 寫下觀察 → 窗口走完後回來驗證。

### 日期問盤

當使用者帶著「某一天是否適合行動」進站，產品不回覆買進或賣出。它把問題改寫成一條可驗證的旅程：

1. 選定目標日、當下情境與 5 / 20 / 60 個交易日歷史對照期。
2. 依交易所公告日曆校正觀測日，任何順延都要明示。
3. 「象」只描述 3° 內主要相位的時間質地，不指定方向。
4. 「證」只比對 `行運星體 + 相位 + 本命星體` 完全相同的歷史窗口。
5. 「事」核對已接入的官方公司事件，並把未覆蓋資料直接列出。
6. 「界」說清楚未還原價格、樣本依賴、命盤精度與非因果。
7. 「記」保存理由、反證條件、回看日期與當時資料日期。

少於 5 筆完整樣本時只顯示案例並標示「樣本不足」。達到門檻也只是描述統計，不轉成勝率、信心分數或行動結論。

### 雙鏡模式

- **玄覽**：公司命格、時間質地、天象主題與儀式式敘事。
- **驗證**：歷史出現次數、全部結果、D+1 / D+5 / D+20、相對大盤、樣本數與失敗案例。

不使用單一「吉／凶」或「買／賣」分數。若需要摘要，可使用「天象張力」，只表示同時發生的重要行運多寡，不表示價格方向。

## 6. 資訊架構

MVP 用一個公司工作台完成主要旅程：

1. **尋公司**：搜尋與公開案例。
2. **定生辰**：成立／法人註冊／首次公開交易／重新上市，附來源、時區與精度。
3. **觀象**：一句摘要、命格印記、本命行星、當期主題。
4. **盤勢**：價格主圖、行運軌、公司事件軌、價格轉折軌。
5. **應驗**：歷史相似配置、結果分布、成功與失敗案例。
6. **憑據**：日期來源、價格資料、算法、時區、更新時間與使用界線。

## 7. 命盤基準與精度

必須把三個概念拆開：

- `命盤基準`：用哪個誕生事件起公司本命盤。
- `探索時間`：現在正在看哪一天或區間。
- `市場事件`：該期間實際發生的財報、拆股、併購、管理層與產品事件。

精度標籤要直接說人話：

- `精確`：時分與地點有可查來源。
- `推定`：採交易所開盤時間。
- `日期`：缺少時分，宮位與上升不解讀。
- `爭議`：來源不一致，可切換並排比較。

> 這張盤只有日期。行星仍可計算，但宮位與上升保持沉默。

## 8. 視覺與互動：東方觀象所

視覺關鍵字是「墨、紙、銅、印」，不是泛用紫色星空、霓虹或水晶球。

- 墨黑 `#141512`
- 米紙 `#F4F0E6`
- 骨白 `#FBF8EF`
- 朱砂 `#B64236`：只用於印章與關鍵節點，不等於下跌。
- 青玉 `#2F7163`
- 古金 `#AD8B4F`
- 水藍 `#3F6175`

圖表區保持純色、清楚網格與高對比數字。紙張顆粒只用在首頁、命格印記與起盤儀式，不蓋在價格圖上。

動效原則：

- 第一次起盤 0.8 秒，行星軌道描線完成，價格線再淡入。
- 存盤時朱砂印章落下一次，不持續閃爍。
- 重要節點只做一次墨暈擴散。
- 支援 `prefers-reduced-motion`。
- 禁止 confetti、粒子爆炸、輪盤、卡包揭曉、價格跳動音效與用紅綠暗示天象好壞。

## 9. 回訪與分享

分享卡保留公司、日期、命盤基準與閱讀鏡的 deep link，並固定帶「象徵性探索・非投資建議」。

回訪不用交易 streak，而是「觀測簿」：

1. 重要窗口前 3 天：「結構主題即將變得明顯，要先記下你的觀察嗎？」
2. 窗口結束：「這個觀測窗口已走完，回來看盤面如何回應。」
3. 每週一封「本週盤勢」，只包含收藏公司的重大窗口與上週回顧。

## 10. MVP、下一版與付費邊界

### MVP

- 臺股上市公司搜尋。
- 成立／上市雙命盤。
- 公司命格印記。
- 日線與主要行運 overlay。
- D+5 / D+20 歷史回看。
- 玄覽／驗證雙鏡。
- 觀察簿、未來窗口與 deep-link 分享。
- 日期問盤與非交易日校正。
- 同組態 D+5 / D+20 / D+60 未還原收盤價變動與完整案例。
- 官方公司事件核對與資料缺口狀態。
- 本機決策日誌與回看日期。
- 來源、精度、方法與免責。

### 下一版

- 加權指數基準與相對報酬。
- 法說、財報、併購、CEO 更換與產品事件軌。
- 成立／註冊／上市多命盤並排。
- 到期提醒與事後回顧。
- 全球市場資料適配器。

### Freemium

- **免費**：公司命格、預設首交易命盤、1Y 價格、每月有限次完整回看、3 家收藏、分享卡。
- **觀星版**：無限收藏、MAX 歷史、窗口提醒、公司比較。
- **研盤版**：完整 event study、相對基準、資料匯出、自訂事件與多命盤研究。

付費賣的是深度與工作流，不是更篤定的預測。資料來源、時間精度、算法、不確定性與免責永遠不放進付費牆。

## 11. 信任與產品邊界

免責不能只放頁尾。它應在決策語境改變時出現：

- 起盤前：事件來源與時間精度。
- 首次解讀下方：「為什麼這樣讀」。
- 切到未來日期：「前方不是答案，而是觀察窗口」。
- 分享卡：「象徵性探索・非投資建議」。
- 設定提醒或付費前：再次說明不是預測服務。
- 方法頁：公開星曆、時區、價格調整語義與事件挑選規則。

應避免：買／賣／持有、目標價、部位比例、進出場點、今日最旺股票、依個人命盤推薦個股、假精準的「93% 準確率」與以免責包裝實際交易指示。

盤勢應使用的語言是：「事件窗口」、「張力」、「歷史相似配置」、「樣本不足」與「沒有顯著關聯」。

台灣正式上線前應由證券法規顧問審查功能與文案。參考金管會對一般理財網站與有價證券分析／推介建議的[函釋](https://law.fsc.gov.tw/LawContent.aspx?id=GL001365)。

## 12. 成功指標

- 首次搜尋到看懂第一個事件的時間 < 90 秒。
- 起盤後事件點擊率。
- 使用者是否切換命盤基準並查看來源。
- 玄覽 → 驗證的轉換率。
- 存盤／記下觀測窗口的比率。
- 窗口結束後的回訪率。
- 分享 deep link 的開啟率。
- 來源、精度與免責不應因轉換率實驗而被隱藏。
